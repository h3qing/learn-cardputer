## 硬件原理

### 声音怎么变成数字:PCM

麦克风前的空气压强是连续变化的,但计算机只认离散的数。PCM(Pulse Code Modulation)的做法简单粗暴:**每隔固定时间量一次电压,把它记成一个整数**。就像用连拍拍一个跳舞的人——单张照片是静止的,但每秒 44100 张连起来,舞蹈就"活"了。

两个参数决定了录像的保真度:

- **采样率**(sample rate):每秒量几次。奈奎斯特定理说,想还原频率为 f 的信号,采样率至少要 2f。人耳上限约 20kHz,所以 CD 选了 44.1kHz——刚好留了点余量。我们的口袋钢琴用 44100Hz 就绰绰有余。
- **位深**(bit depth):每次量的数用几个 bit 存。16-bit 意味着幅度被切成 65536 级(-32768 到 +32767),级数越多,量化噪声越小。

所以"一秒钟的 16-bit 单声道 44.1kHz 音频"就是 44100 个 `int16_t`,共 88200 字节。声音,不过是一个数组。

### I2S:按拍子搬数字的传送带

数组躺在内存里不会响,得有人**以精确的节奏**把样本一个一个送到功放。这就是 I2S(Inter-IC Sound)总线的活。它和第 2 课的 SPI 长得像(也是时钟 + 数据),但灵魂不同:SPI 是"有事说事"的突发传输,I2S 是**永不停歇的节拍器**——每个样本必须准时送达,早了晚了都会让声音变形。

三根线各司其职:

```
BCLK  ┐_┌┐_┌┐_┌┐_┌┐_┌┐_┌┐_┌┐_┌   位时钟:每跳一下,搬运 1 个 bit
LRCLK ______________┌──────────   声道选择(WS):低电平=左声道,高电平=右声道
DATA  <---左声道 16 bit---><---右声道 16 bit--->
```

- **BCLK**(bit clock):最快的心跳,每个上升沿传 1 bit;
- **LRCLK**(又叫 WS,word select):慢心跳,它翻转一次就切换一次声道,它的频率就等于采样率;
- **DATA**:样本的比特流本体。

三者是严格的倍数关系:`BCLK = 采样率 × 位深 × 声道数`。44.1kHz × 16-bit × 2 声道 = 约 1.41MHz。你可以把 LRCLK 想成鼓点,BCLK 想成鼓点里的十六分音符——整个乐队(发送方和接收方)靠它们对齐,不需要任何"开始/停止"标记。

### NS4168:数字进,声波出

Cardputer 的喇叭由 NS4168 驱动。传统链路是"数字 → DAC 变模拟 → 模拟功放放大",而 NS4168 是**数字输入 D 类功放**:它直接吃 I2S 数字信号,在芯片内部把 PCM 变成高频 PWM 开关波形去推喇叭,再靠喇叭自身的电感特性滤掉高频开关成分,剩下的就是你要的声波。

好处很实际:全程数字传输,不怕板子上的电源噪声串进模拟信号(否则你会听到 WiFi 工作时喇叭"滋滋"叫);D 类功放的晶体管只在"全开/全关"两个状态,发热极小,效率轻松上 85%+,特别适合电池设备。ESP32-S3 因此**根本不需要音频 DAC**——I2S 三根线直连功放,链路干净利落。

### DMA:CPU 不搬砖

44.1kHz 意味着每 22.7 微秒就要送一个样本。如果让 CPU 亲自搬,它就别干别的了——屏幕不用刷,键盘不用扫。解法是 **DMA**(Direct Memory Access):CPU 把一整块样本缓冲写进内存,告诉 DMA 控制器"这块,搬去 I2S 外设",然后就撒手不管。DMA 像餐厅的传菜机器人,厨师(CPU)只管炒下一锅菜(算下一段波形),送菜完全自动。

I2S 驱动通常配**多块 DMA 缓冲轮转**:DMA 在播第 1 块时,CPU 填第 2 块。只要 CPU 填得比 DMA 播得快,声音就绵延不断;一旦没填上(缓冲欠载),喇叭会重复旧数据或输出静音,你会听到"咔哒"或断续——这是本课挑战任务里最常见的 bug。

### 波形形状 = 音色

同样弹一个 A4(440Hz),钢琴和小提琴为什么听起来不一样?因为**波形形状不同**,也就是泛音(谐波)成分不同:

- **正弦波**:只有基频,没有任何泛音。声音"纯净空灵",像音叉;
- **方波**:含所有奇次谐波(3f、5f、7f...),按 1/n 衰减。声音"亮而冲",就是红白机、Game Boy 的经典音色;
- **锯齿波**:奇偶次谐波全都有。最"厚",是模拟合成器弦乐和铜管音色的基底。

本课你会亲手写出这三个公式,然后闭着眼分辨它们——从此听芯片音乐时,你能报出用的是哪种波。

## 动手实验

### 第 0 步:确认发声通路

新建 PlatformIO 工程(`board = m5stack-stamps3`,依赖 `m5stack/M5Cardputer` 库,配置同第 2 课)。先用库自带的 `tone()` 确认硬件没问题:

```cpp
#include <M5Cardputer.h>

void setup() {
    auto cfg = M5.config();
    M5Cardputer.begin(cfg);
    M5Cardputer.Speaker.setVolume(128);   // 音量 0~255,别一上来就拉满
    M5Cardputer.Speaker.tone(440, 500);   // 440Hz(A4),响 500ms
}

void loop() {}
```

烧录后应该听到"哔"一声。这一步的意义:M5Cardputer 库已经封装好了 I2S 引脚配置和 NS4168 的初始化,你不用查引脚就能出声(具体引脚见文末官方 pinout)。`tone()` 内部其实就是在生成方波样本喂给 I2S——接下来我们自己干这件事。

### 第 1 步:手工合成一段正弦波

`tone()` 是别人合成好的,现在自己算样本:

```cpp
#include <M5Cardputer.h>
#include <math.h>

static constexpr int   SAMPLE_RATE = 44100;   // 每秒采样数
static constexpr float FREQ        = 440.0f;  // A4 音高
static constexpr int   N           = SAMPLE_RATE;  // 1 秒的样本数

int16_t wave[N];

void setup() {
    auto cfg = M5.config();
    M5Cardputer.begin(cfg);
    M5Cardputer.Speaker.setVolume(128);

    // 逐个样本计算:第 i 个样本对应时刻 t = i / SAMPLE_RATE
    for (int i = 0; i < N; i++) {
        float t = (float)i / SAMPLE_RATE;
        float s = sinf(2.0f * M_PI * FREQ * t);   // -1.0 ~ +1.0
        wave[i] = (int16_t)(s * 16000);           // 缩放到 16-bit 范围(留余量防削波)
    }

    // 把整块 PCM 缓冲交给 Speaker,底层由 I2S+DMA 按 44100Hz 定时送出
    M5Cardputer.Speaker.playRaw(wave, N, SAMPLE_RATE, false /*单声道*/);
}

void loop() {}
```

听到一秒钟"纯"的 A4。**为什么缩放到 16000 而不是 32767?** 留出动态余量——之后多个音叠加(和弦)时,顶到满量程会削波(clipping),那是很难听的破音。

### 第 2 步:换波形,听音色

只换掉计算 `s` 的那一两行,其余不动(下面是两种可选公式,一次用一种):

```cpp
// 方波:正弦为正取 +1,为负取 -1 —— 游戏机音色
float s = (sinf(2.0f * M_PI * FREQ * t) >= 0) ? 1.0f : -1.0f;

// 锯齿波:相位从 0 匀速涨到 1,瞬间跳回,映射到 -1~+1
float phase = fmodf(FREQ * t, 1.0f);
float s = 2.0f * phase - 1.0f;
```

分别烧录,闭眼听三遍。同一个 440Hz,正弦"呜"、方波"哔"、锯齿"滋"(带电流感的扎耳)——频率没变,变的只是形状。这就是"音色 = 波形"的第一手证据。

### 第 3 步:相位累加器——实时合成的钥匙

第 1 步预先算好整段缓冲,但钢琴按键时长不可预知,必须**实时**逐块合成。工业标准做法是相位累加器:不再用绝对时间 t,而是维护一个 0~1 的相位,每个样本前进 `freq / SAMPLE_RATE`:

```cpp
float phase = 0.0f;
float phase_inc = FREQ / SAMPLE_RATE;   // 每个样本相位前进多少

void fillBlock(int16_t* buf, int n) {
    for (int i = 0; i < n; i++) {
        buf[i] = (int16_t)(sinf(2.0f * M_PI * phase) * 16000);
        phase += phase_inc;
        if (phase >= 1.0f) phase -= 1.0f;   // 回卷,永不溢出
    }
}
```

**为什么必须这样?** 一是 float 的绝对时间 t 越涨精度越差,几分钟后音高会飘;二是改变 `phase_inc` 就能瞬间变调而相位连续——切音符时不会"咔哒"。在 `loop()` 里反复调 `fillBlock` 再 `playRaw` 一小块(如 512 样本),你就有了一台最小的实时合成器。注意两个坑:一,`playRaw` **不会拷贝你的数据**,播放期间缓冲必须保持有效——所以要准备两块缓冲交替干活(填 A 时播 B,再填 B 播 A,行话叫 ping-pong 缓冲);二,M5Unified 的 Speaker 内部有多个虚拟声道,`playRaw` 的 `channel` 参数默认 -1 表示"自动挑一个空闲声道"——连续调用会被分到**不同声道同时混音**,而不是一块接一块播。流式播放必须显式传同一个声道号(如 `playRaw(buf, n, SAMPLE_RATE, false, 1, 0)`,最后的 0 就是声道),这样第二块会排队等第一块播完,无缝衔接。只要你产出样本的速度跟得上消耗,声音就无缝。

## 挑战任务

**造一台口袋钢琴**——下次朋友聚会,从兜里摸出这台小机器,当场即兴一段 8-bit 版《欢乐颂》,没人忍得住不问"这是什么?":键盘第三排(a s d f ...)当白键、第二排当黑键,按下发声、抬起停止;屏幕画一排琴键,按到哪个哪个亮起来跳动;按数字键 1/2/3 切换正弦/方波/锯齿波音色。**验收线:同时按三个键出和弦不破音;按键到出声无可感延迟(< 20ms 左右)。**

不给完整代码,给路线图:

1. **音高表**。十二平均律:`f = 440 × 2^((n-69)/12)`,n 是 MIDI 音符号(A4 = 69)。先造一个"键盘字符 → 频率"的映射表,串口打印验证。
2. **单音钢琴**。用第 3 步的相位累加器,把 `M5Cardputer.Keyboard` 的按键事件(第 3 课的老朋友)接到 `phase_inc` 上:按下设频率、抬起静音。先别管画面,让一个键能弹。
3. **复音(polyphony)**。声部结构体数组(比如 4 个 voice),每个有自己的 phase、freq、是否活跃;每个样本 = 所有活跃 voice 输出**求和**。关键坑:4 个 ×16000 幅度的音叠加必然溢出 int16 —— 想想怎么办(每个 voice 除以声部数?还是求和后再缩放并夹紧?两种做法音量表现不同,都试试)。
4. **音色切换**。把波形公式抽成一个函数指针或 switch,数字键切换。留意:切换时相位要不要重置?不重置会怎样?
5. **琴键动画**。第 4 课的游戏循环直接搬来:输入-合成-渲染分离,sprite 双缓冲画琴键。**警告:渲染千万不能阻塞音频填充**——如果整帧 blit 花太久,DMA 缓冲欠载,声音就"咔哒"。思路:每帧只填一小块音频、渲染放在音频填充之后、或者加大 Speaker 的缓冲。这个矛盾就是本课最有嚼头的工程问题。
6. **炫耀彩蛋**(选做):内置一段硬编码的旋律数组(音符 + 时长),按反引号(`` ` ``)键自动演奏,方波音色,瞬间还原红白机 BGM。录屏发朋友圈。

## 深入一层

- **削波实验**:故意把幅度拉到 40000 再转 int16_t,听听溢出回卷(wrap-around)的声音有多惨烈;再改成夹紧(clamp 到 ±32767),对比两种"破音"的区别。你会理解为什么音频代码里到处是 `constrain`。
- **ADSR 包络**:真钢琴的音量是"敲击瞬间最响,然后衰减"。给每个 voice 加一个随时间变化的幅度系数(Attack-Decay-Sustain-Release 四段折线),口袋钢琴立刻从"电子哔哔"变成"有手感的乐器"。
- **查表代替 sinf()**:每个样本调一次 `sinf()` 挺贵。预计算一张 1024 点的正弦表,用相位直接索引(这正是 DDS 直接数字频率合成的原理)。测测 CPU 占用降了多少——第 6 课做实时音频处理时,这个优化会救你的命。
